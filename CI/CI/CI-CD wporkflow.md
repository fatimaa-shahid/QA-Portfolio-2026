# CI/CD Workflow for QA (Selenium Automation)

## Introduction
CI/CD (Continuous Integration / Continuous Deployment) is a process used to automate software testing and delivery.

## Workflow
- Developer pushes code to GitHub repository
- GitHub Actions or Jenkins automatically triggers pipeline
- Selenium automation scripts are executed
- Test reports are generated automatically
- If tests pass, code moves to deployment stage

## QA Role
- Ensure quality before deployment
- Run automated test cases
- Detect bugs early in development cycle

## Tools Used
- GitHub Actions
- Jenkins
- Selenium WebDriver

## Conclusion
CI/CD helps in faster, reliable, and automated software delivery.
